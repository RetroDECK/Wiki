**Added:**

- **BACKUP:** Simplified the backup prompt and choices.
- **COMPRESSION:**  CHDMAN now also checks: arcade, atomiswave, cdimono1, cdtv, fmtowns, megacdjp, mess, naomi, neogeocdjp as valid compression targets.

- **FRAMEWORK:** Added a verification step to ensure Steam Input controller layout files are always injected during post-update.
- **FRAMEWORK:** Migrated PortMaster and SRM data into their dedicated component files.
- **FOLDER ICONS:** Added a folder icon for PrimeHack.
- **PRIMEHACK:** Separated the PrimeHack component and added its own folder in `roms/primehack` based on user feedback.

**Changed:**

- **FRAMEWORK:** Updated the backup process to exclude log files, reducing the risk of corruption during writes.

## Components

<img src="../../../wiki_icons/retrodeck/icon-component.svg" width="75">

**Fixed:**

- **AZAHAR:** Azahar's configuration files contained several errors in hot-key mappings and other settings, warranting a configuration reset.
- **AZAHAR:** Fixed swap A/B X/Y statement error.
- **GENERAL:** Fixed spelling errors across several component-ingredient files.
- **PCSX2:** Corrected a `retrodeck/mods/pcsx2` folder-path issue in the backup function.
- **PICO-8:** PICO-8 is now looking in the correct path and is working again. <img src="../../../wiki_images/logos/pico-8-logo-alt.png" width="15"> üõñ
- **PORTMASTER:** RetroDECK now sources the latest fix from the PortMaster Team, which should resolve the current issues.
- **PRIMEHACK:** PrimeHack's configuration files contained several errors in hot-key mappings and other settings, warranting a configuration reset.
- **RETROARCH:** Corrected a `retrodeck/bios/mame2003-plus` folder-path issue in the backup function.
- **SRM:** Fixed an issue where RetroDECK controller layouts were not applied under certain conditions due to versioning errors.

**Changed:**

- **ES-DE:** PrimeHack has been removed as an alternative emulator for GameCube and Wii, and now has its own PrimeHack system, linked to the `roms/primehack` folder.
